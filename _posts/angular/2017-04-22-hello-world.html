Angular en Docker

docker-compose run -p 0.0.0.0:3000:3000 app_node bash

Nuevo proyecto

{% highlight ruby linenos %} {% raw %} ng new angular-project cd angular-project ng serve --host 0.0.0.0 --port 3000 {% endraw %} {% endhighlight%}

Modificando el archivo package.json podemos configurar en start el host y puerto por defecto:

{% highlight javascript linenos %} {% raw %} { "name": "angular-project", "version": "0.0.0", "license": "MIT", "scripts": { "ng": "ng", "start": "ng serve --host 0.0.0.0 --port 3000", "build": "ng build", ... ... {% endraw %} {% endhighlight%}

Ejecutar npm start para levantar el servicio de angular

Crear componente

  1. Modificamos el fichero .angular-cli.json para evitar crear plantillas y estilos extras. {% highlight javascript linenos %} {% raw %} "defaults": { "styleExt": "css", "component": { "inlineStyle": true, "inlineTemplate": true } } {% endraw %} {% endhighlight%}
  2. Crear carpeta que contendrá los componentes. {% highlight javascript linenos %} {% raw %} cd src/app mkdir components {% endraw %} {% endhighlight%}
  3. En la raíz del proyecto generamos el componente. {% highlight javascript linenos %} {% raw %} ng g c components/my-new-component {% endraw %} {% endhighlight%}
  4. En app.module.ts se importará automáticamente el componente generado. {% highlight javascript linenos %} {% raw %} import { AppComponent } from './app.component'; import { MyNewComponentComponent } from './components/my-new-component/my-new-component.component'; @NgModule({ declarations: [ AppComponent, MyNewComponentComponent ], imports: [ BrowserModule, ... ... {% endraw %} {% endhighlight%}
  5. En nuestro componente modificamos la plantilla: {% highlight javascript linenos %} {% raw %} @Component({ selector: 'app-my-new-component', template: `

    Hola Angular!

    `, styles: [] }) {% endraw %} {% endhighlight%}
  6. Agregamos el selector al componente base: {% highlight html linenos %} {% raw %}

    {{title}}

    {% endraw %} {% endhighlight%}
